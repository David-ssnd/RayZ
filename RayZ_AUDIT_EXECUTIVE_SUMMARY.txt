📍 ESP32 FIRMWARE (esp32/) - 4 CRITICAL ISSUES

  1. ID Generation Bug (game_state.cpp:139)
     ❌ rand_u8() generates 0-255, should be masked to limits
     ⚠️  Impact: 75% of device_ids invalid, 87.5% of player_ids invalid
     📝 Fix: Mask with 0x3F for device_id, 0x1F for player_id

  2. NVS Load No Validation (game_state.cpp:87,92)
     ❌ Loads stored IDs without checking ranges
     ⚠️  Impact: Corrupted IDs persist across reboots
     📝 Fix: Mask loaded values before storing in config

  3. WebSocket Config Update No Validation (ws_server.cpp:153,156)
     ❌ Direct assignment of JSON integer values
     ⚠️  Impact: Malicious clients can inject device_id:500, player_id:200
     📝 Fix: Validate ranges (0-63 for device, 0-31 for player) before accepting

  4. NVS Save No Validation (game_state.cpp:120-121)
     ❌ Saves without verifying valid ranges first
     ⚠️  Impact: Creates permanent corruption in flash storage
     📝 Fix: Mask before saving to NVS

📍 WEB DATABASE (web/packages/database/) - 3 CRITICAL ISSUES

  5. Prisma Device.deviceId (schema.prisma:81)
     ❌ Stored as unconstrained Int
     ⚠️  Impact: Database accepts values up to 2 billion
     📝 Fix: Add @@check("deviceId >= 0 AND deviceId <= 63")

  6. Prisma Player.number (schema.prisma:131)
     ❌ Unconstrained Int despite comment saying 0-31
     ⚠️  Impact: Database violates protocol specification
     📝 Fix: Add @@check("number >= 0 AND number <= 31")

  7. Prisma Team.number (schema.prisma:111)
     ❌ Unconstrained Int despite comment saying 1-31
     ⚠️  Impact: Invalid values accepted and propagated
     📝 Fix: Add @@check("number >= 0 AND number <= 31")
